# Приложение «Тест»

Android‑приложение на Kotlin (XML), в котором пользователь проходит тест. Данные берутся из `assets` через имитацию сетевого слоя, чтобы можно было показать обработку загрузки и ошибок.

## Быстрый запуск

1. В Android Studio запустите конфигурацию `app`.
2. Дождитесь сборки проекта.
3. На первом экране приложения нажмите кнопку "Начать тестирование".

Минимальная версия SDK: 24.

## Что внутри и почему так

- **Чистая архитектура**: разделение на `data`, `domain`, `presentation`, `di` — чтобы было проще ориентироваться и тестировать отдельные части.  
- **Имитация API**: данные читаются из `assets/questions_movie_valid.json` с искусственной задержкой — так можно продемонстрировать `loading / error / content`.
- **Сохранение прогресса**: ответы сохраняются в Room, прохождение не сбрасывается при повороте.  
- **Навигация**: `Navigation Component` между тремя экранами.

## Технологии

- Kotlin
- Android SDK (XML)
- Kotlin Coroutines
- Kotlin Flow
- Room
- Navigation Component
- ViewModel
- Gradle (Kotlin DSL)

## Спорные/упрощённые моменты

- **Одна ViewModel на весь флоу теста**. Это упрощает обмен данными между экранами, но в долгосрочной перспективе может быть логичнее разделить по экрану.
- **Излишнее разделение по слоям**. Сделано с целью продемонстрировать понимание чистой архитектуры.

## Экраны

1. Карточка теста — описание и кнопка запуска.
2. Прохождение теста — вопросы с вариантами ответа.
3. Результат — количество правильных ответов и баллы (порог 75%).
